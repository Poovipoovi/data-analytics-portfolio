# Day 3 SQL Practice

Today I revised and learned advanced SQL filtering concepts in detail.

I studied the `LIKE` operator, which is used in a `WHERE` clause to search for a specific pattern in a column’s text data.

I also learned different SQL wildcards used for pattern matching:

- `%` → represents zero, one, or multiple characters  
- `_` → represents a single character  
- `[ ]` → represents any single character within the bracket  
- `^` → represents any character not in the bracket  
- `-` → represents a range of characters within a specific range  
- `{ }` → represents escaped characters  
- `*` → represents zero or more characters  
- `?` → represents a single character  
- `!` → represents any character not in the bracket  
- `#` → represents any single numeric value  

I learned how wildcard placement changes the search result, such as starts with, ends with, contains, and exact position matching.

I studied the `IN` operator, which checks whether a specified value matches any value in a provided list.

I also learned `NOT IN`, which returns records that do not match any values in the list.

Finally, I understood how `IN` can be used with subqueries, allowing the main query to return results based on another query’s output.

Day 3 strengthened my knowledge of SQL filtering and pattern-based searching.
